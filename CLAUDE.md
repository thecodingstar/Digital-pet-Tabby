# CLAUDE.md

Guidance for Claude Code (claude.ai/code) working in this repo.

## Project: Digital Pet Tabby — a living taskbar cat

Tabby is an animated pixel cat that lives on the Windows taskbar. She reacts to
Claude Code activity, runs an autonomous behaviour engine with drives when idle,
talks via an LLM, and grows a persistent personality + memory that increasingly
runs offline.

**Status:** ✓ Working — sprite mascot + autonomous brain + drives + LLM voice +
self-improving offline memory + interactive get-to-know-you quiz + brain dashboard.

## Run

```bash
# one-click setup + launch (fresh clone): finds/installs Python, installs
# PyQt5, creates .env, then launches minimized. Double-click or:
start_mascot.bat
# auto-restarting launcher (recommended for dev) — watches sources, restarts:
python run_mascot.py
# or just the mascot once:
python taskbar_mascot_cat.py
```

Only external dep is **PyQt5** (`requirements.txt`); everything else is stdlib.
Tabby runs without an API key (canned offline lines); a free Groq key in `.env`
gives her a live voice — see `README.md` / `.env.example`.

**Interpreter note:** the machine's `python3.15.exe` is a broken uv shim. Use the
real CPython (`...\Programs\Python\Python314\python.exe`). `run_mascot.py`
auto-discovers a working PyQt5 interpreter, so prefer launching through it.

## How Tabby works

### Two modes (loosely coupled — her brain always runs)
- **brain** (default) — `brain.py` drives behaviour autonomously; `brain.tick`
  runs every tick regardless of Claude.
- **react** — Claude only commandeers the animation when she *chooses* to watch
  your work (episodic: on each work-start she rolls `p≈0.2+0.5·curiosity+social/300`
  for a 6–16s window — `WORK_STATES`), or for a brief `RESULT_REACT_S` reaction to
  a result event (`RESULT_STATES`). Urgent drives always override. So she mostly
  does her own thing while you code, glancing over when she feels like it.

### State source
Reads `~/.claude/statusline/state/<session>.json` (written by
`statusline/state_writer.py`). 600s staleness cutoff. Emitted states: idle,
thinking, tool_running, subagent_running, question, tool_success, tool_failure,
permission, done, auth_success.

### Drives + behaviour learning (brain.py)
`energy/hunger/social/fear` drift over time; urgent drives override behaviour
(fear→cower, hunger→beg, social→seek). On top of that: **mood inertia** (smoothed
valence/arousal), **behaviour affinity** (learns which behaviours earn your
attention), **trust + jumpiness** (consoling calms, error-storms sensitize), and a
**user-activity rhythm** (naps cluster in your quiet hours). Idle adds flavour
behaviours (`loaf/knead/ponder/watch`) + a cursor-proximity glance, and the brain
tallies **behaviour usage** (`behavior_counts`/`behavior_secs`) to expose
repetition. **Cross-brain wiring (X-series):** the mascot pulls
`cat.behavior_hints()` → `brain.apply_hints()` so learned traits/quiz prefs bias
behaviour selection + scare; Claude transitions feed `brain.note_activity()` so her
decaying activity rhythm lets her **anticipate** sessions (`pre_active` →
`anticipate` behaviour). One-directional seam; see BRAIN.md §3.5. All persist.

### Interactions
- **Left-click** = pet (consoles if scared). **Click vs drag** is decided on
  release via a 6px deadzone (`mousePressEvent`/`mouseMoveEvent`/`mouseReleaseEvent`).
- **Left-drag** = reposition her along the taskbar; on release she resumes free
  roaming from the new spot. A fast yank startles her (Phase-4 `drag` trigger);
  dragging is suppressed while a quiz card is up. Position is not persisted.
- **Right-click** = menu: Ask me something (force a quiz) / Feed / Pet / Sleep /
  📊 Stats (build+open the dashboard) / Quit.
- **Hover** = info panel (what she's doing + drive bars + bond %).

### Bond decay (chatter.py, Phase 1b)
Affection is no longer monotonic. A genuine attention event (pet/feed/console/quiz
answer) stamps `last_attention`; after a grace window (`BOND_GRACE_H`) of neglect it
**cools** at `BOND_DECAY_PER_DAY`, loyalty-damped (higher tiers slower) and floored
at `BOND_FLOOR` (≥ the quiz gate). Computed on load + on a 60s worker tick, so it
works across restarts and while closed. `save()` must NOT touch `last_attention`.

### Offline-first (chatter.py, Phase 1)
A single `online` health flag flips at the one network choke point (`_post_chat`):
`NET_FAIL_MAX` consecutive failures → offline + a re-probe cooldown; success
recovers. `_llm`/`_llm_question`/`reflect` gate on `_net_ready()`. Offline, the quiz
draws from `questions_library.json`, whose per-option tags still teach traits/prefs/
facts — so she keeps learning with no API. `local_question_rate`/`offline_flips`
land in metrics.

### Expanded fear (taskbar_mascot_cat.py + brain.py, Phase 4)
`_fear_trigger()` maps Claude transitions to `brain.scare(amount, trigger=…)`:
`tool_failure`, `error_storm` (burst of failures), `permission` (escalating),
`activity_spike` (after long idle). Plus `neglect` (very long no-attention) and
`drag` (fast yank). All trust/shyness/comfort-damped in `scare()`.

### Get-to-know-you quiz (chatter.py + QuestionBubble)  — see BRAIN.md
She asks short questions with clickable answers, at random in any mode
(cooldown-gated). API-generated for uniqueness when online; offline (or out of
budget) she pulls an unused question from the **tagged `questions_library.json`**
(200+ items), tracking used ids via the `user_profile` keyset so nothing repeats
until the bank is exhausted. Answered API questions are appended to
`learned_questions.json` for offline reuse. Each option carries `traits` (nudge her
personality), structured `prefs` (facts about you, read by `behavior_hints`), and an
optional `fact`. Answers fill `user_profile`, nudge traits, +1 affection, and feed
the system prompt + musing cadence. `maybe_ask`/`ask_now`/`poll_question`/
`answer_question` mirror say/poll.

### Voice + self-improving memory (chatter.py)  — see BRAIN.md for full detail
- LLM via any OpenAI-compatible endpoint (default Groq `llama-3.3-70b-versatile`);
  needs a browser `User-Agent` header (set).
- **Structured context recall** (`RECALL_MODE`, default `structured`; `vector`
  legacy fallback): typed ctx + weighted field-match similarity, zero deps.
- **Reward-scored lines** — outcomes (pet/feed=good, scare=bad) EWMA each line;
  recall weights `(sim)³·(0.5+reward)·anti_repeat·cross_event`. Quality eviction.
- **Context-aware coverage + pool maturity** drive `local_prob` (M8) → accumulated
  lines actually get recalled; per-event floors; served-repeats lose reward;
  anti-repeat ring 16. Verified ~3× local-hit-rate, ~½ API in `sim_harness.py`.
- **Confidence facts** (categories, evidence, decay) from reflection.
- **Daily budget** 600; **telemetry** → `cat_metrics.json`. All network on a
  worker thread (RLock: prompt under lock, network outside, store under lock).
- Guardrails: ASCII-only `_clean_line`, dedup, UTF-8 + atomic writes, canned
  fallback, schema-versioned migration + self-heal on load.

## Conventions / gotchas
- Window stands ON the taskbar's top edge (`FOOT_OVERLAP`), forced topmost via
  `SetWindowPos(HWND_TOPMOST)`, re-asserted every ~2s.
- Sprites scaled with `Qt.FastTransformation` (nearest) to stay crisp.
- `cat4_states/*.png` ARE tracked: `.gitignore` has a `*.png` rule then a
  `!cat4_states/*.png` exception. New sprites land in git automatically; don't
  re-add the blanket ignore.
- `archive/` is gitignored entirely — old experiments/scratch stay local-only.
- Don't launch via the `python3.15.exe` uv shim; it can't spawn children.

## Dashboard
`python dashboard.py [--open]` (or right-click → 📊 Stats) reads
`cat_metrics/state/brain.json` and writes a standalone `dashboard.html` (inline
SVG, no server/CDN, stdlib only): memory-quality + API-reliance trends, an
"improving?" verdict, traits, behaviour affinity, **behaviour usage** (over-used
states >25% flagged red), active-hours rhythm, confidence facts, quiz answers,
learned-lines-per-event.
