# Digital Pet Tabby 🐈

An animated pixel-cat mascot that perches on the **Windows taskbar**. She reacts
to [Claude Code](https://claude.com/claude-code) activity when it's running,
wanders and naps on her own when it isn't, talks through a free LLM, and grows a
persistent personality + memory over time.

- **Lives on the taskbar.** Frameless, always-on-top, transparent — her feet rest
  on the taskbar's top edge.
- **Two modes.** *React* — mirrors Claude Code's real state (thinking, running
  tools, success, failure, waiting). *Brain* — when idle she runs an autonomous
  behaviour engine with drives (energy / hunger / social / fear).
- **She talks.** Speech bubbles generated by an OpenAI-compatible LLM (free Groq
  by default), with a self-improving offline memory so she works without network.
- **You interact.** Left-click = pet · Right-click = menu (Feed / Pet / Sleep /
  Quit) · Hover = info panel (mood, drives, bond %).

## Quick start (Windows)

```text
1. Clone this repo (or download the ZIP and extract it).
2. Double-click  start_mascot.bat
```

That's it. On first run `start_mascot.bat` will:

1. **Find Python** — and if it's missing, install it via `winget`
   (or open python.org so you can install it yourself).
2. **Install PyQt5** (the one dependency).
3. **Create a `.env`** from `.env.example` so Tabby is ready for a key.
4. **Launch** the auto-restarting mascot, minimized.

Tabby appears on your taskbar. To stop her: right-click → **Quit**.

### Manual run (any OS with a taskbar-like setup / for devs)

```bash
pip install -r requirements.txt
python run_mascot.py          # recommended: watches files, restarts on edit/crash
# or run once:
python taskbar_mascot_cat.py
```

## Giving Tabby her voice — free Groq API key

Tabby **runs without a key** (she falls back to built-in offline lines). Adding a
free key lets her generate fresh, personality-driven speech.

1. Make a free account at **https://console.groq.com**
2. Create a key at **https://console.groq.com/keys** (starts with `gsk_…`).
3. Open the **`.env`** file in the project folder (created on first run; otherwise
   copy `.env.example` → `.env`) and paste your key:

   ```env
   GROQ_API_KEY=gsk_your_real_key_here
   GROQ_MODEL=llama-3.3-70b-versatile
   ```

4. Save. Tabby reloads automatically (or restart `start_mascot.bat`).

> **Editing later / rotating a key:** just edit `.env` again. `.env` is gitignored
> and **never committed**, so your key stays private.
>
> **Other providers:** any OpenAI-compatible endpoint works — copy
> `cat_config.example.json` → `cat_config.json` and fill in `base_url` / `api_key`
> / `model` (NVIDIA, OpenRouter, etc. — see comments in that file).

## How she reads Claude Code state (optional)

The *react* mode reads per-session JSON from
`~/.claude/statusline/state/<session>.json`, written by
`statusline/state_writer.py`. Register that as a Claude Code statusline/hook to
make Tabby mirror your live coding activity. **This is optional** — without it she
simply stays in autonomous *brain* mode.

## Project layout

| File / dir | Role |
|------------|------|
| `taskbar_mascot_cat.py` | PyQt5 window: renders sprites on the taskbar, game loop, speech bubble, hover panel, menu, click-to-pet |
| `brain.py` | Autonomous behaviour engine + drives (energy/hunger/social/fear) |
| `chatter.py` | LLM voice, personality, self-improving reward-scored memory |
| `run_mascot.py` | Launcher: finds a working Python, runs the mascot, restarts on edit/crash |
| `start_mascot.bat` | One-click setup + launch (Python + deps + `.env`) |
| `cat4_states/` | The 42 sprite PNGs Tabby is drawn from (+ `MANIFEST.md`) |
| `sim_harness.py` | Offline sim (stubbed LLM) for measuring memory changes |
| `BRAIN.md` | Full brain + memory design — read before changing `brain.py`/`chatter.py` |

### Runtime files (created locally, gitignored)

`.env` (your key) · `cat_config.json` · `cat_state.json` (personality) ·
`cat_brain.json` (learned lines) · `cat_metrics.json` (telemetry). None are
committed — a fresh clone regenerates them.
